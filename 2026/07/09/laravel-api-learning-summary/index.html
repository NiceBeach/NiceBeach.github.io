核心知识点总结

1. 预加载优化

N+1 问题：

  • 查询主模型 + 循环访问关联模型
  • 导致 1 + N 次数据库查询

解决方案：

  • 使用 with() 预加载关联数据
  • 查询次数降低到 2 次
  • 指定字段：with('user:id,name,email')

2. withCount 的巧妙用法

分解：

  • tasks - 关联关系名
  • as completed_tasks - 自定义字段名
  • => function ($query) - 添加筛选条件

作用：

  • 一个查询完成多个统计
  • 避免循环查询
  • 性能优化

3. API 文档自动化

Scribe 优势：

  • 自动分析代码生成文档
  • 支持 PHPDoc 注解增强说明
  • 界面美观、易于测试
  • 导出 Postman 集合和 OpenAPI 规范
